# MRA 模型消融实验与参数敏感性分析方案

## 1. 实验目标

围绕 `mra.py` 中的 AGF-ADNet 模型，完成两类分析：

1. 消融实验：验证各关键模块对持续故障检测性能的贡献。
2. 参数敏感性分析：评估关键训练参数、结构参数和检测后处理参数变化时，模型性能的稳定性。

核心评价任务保持不变：利用训练集重构误差建立阈值，在测试集上进行异常检测，并重点关注持续故障检测指标。

## 2. 统一实验设置

- 数据集：`data/train/train_1.csv`、`data/test/test_C5_1.csv`
- 输入窗口：默认 `seq_len=50`
- 窗口采样：沿用 `mra.py` 当前滑窗逻辑
- 标准化：沿用当前“NaN 置 0 后标准化”的流程
- 随机种子：
  - 消融实验：`(40, 41, 42)` 三个种子做快照集成
  - 训练型敏感性分析：`(40,)` 单种子扫描，控制总体计算量
- 训练轮数：默认 `epochs=3`
- 批大小：`batch_size=32`
- 学习率：默认 `lr=1e-3`
- 检测后处理默认参数：
  - `EWMA alpha=0.02`
  - `threshold_std=1.5`
  - `min_run=150`

说明：

- 为保证对比公平，除当前正在扫描的变量外，其余设置均保持与基线一致。
- 持续故障检测指标作为主指标，点级原始异常检测指标作为辅助指标。

## 3. 评价指标

每个实验统一输出以下指标：

- 原始点级指标：`Accuracy`、`Precision`、`Recall`、`F1`
- 持续故障指标：`Accuracy`、`Precision`、`Recall`、`F1`
- 分数统计：训练集均值/标准差、测试集均值/标准差
- 阈值信息：原始阈值、平滑阈值
- 报警统计：点报警数量、持续报警数量

## 4. 消融实验设计

基线模型记为 `full`，其余实验均只改动一个关键设计。

| 实验名 | 改动内容 | 目的 |
| --- | --- | --- |
| `full` | 完整 AGF-ADNet | 作为参考基线 |
| `no_graph` | 去掉自适应图学习，GCN 使用单位邻接 | 验证图结构学习贡献 |
| `no_gcn` | 时间分支去掉 GCN，仅保留 TCN | 验证图卷积贡献 |
| `no_tcn` | 时间分支去掉 TCN，仅保留 GCN | 验证多尺度时域卷积贡献 |
| `no_freq` | 去掉频域补全分支，仅使用时间分支 | 验证频域建模贡献 |
| `mean_fusion` | 将门控融合改为简单平均融合 | 验证门控融合贡献 |
| `no_sampling_embed` | 去掉采样率类型嵌入 | 验证多采样率位置编码贡献 |
| `recon_only_loss` | 损失函数仅保留重构项，去掉频域损失和图稀疏正则 | 验证联合损失贡献 |

预期分析重点：

- 哪个模块被去掉后持续故障 `F1` 下降最多
- 哪些模块主要影响 `Recall`
- 哪些模块会显著改变误报数量或阈值稳定性

## 5. 参数敏感性分析设计

### 5.1 需要重新训练的参数

采用单因素扫描，其余参数固定为基线值。

| 参数 | 取值 |
| --- | --- |
| `seq_len` | `30, 50, 70` |
| `d_model` | `32, 64, 96` |
| `lr` | `5e-4, 1e-3, 2e-3` |

分析重点：

- 较短/较长窗口对异常分数分离度的影响
- Transformer 隐空间维度是否存在收益饱和
- 学习率变化对稳定性和持续故障指标的影响

### 5.2 无需重新训练的检测参数

复用基线模型分数，仅重新执行检测后处理。

| 参数 | 取值 |
| --- | --- |
| `EWMA alpha` | `0.01, 0.02, 0.05` |
| `threshold_std` | `1.0, 1.5, 2.0` |
| `min_run` | `100, 150, 200` |

分析重点：

- 平滑强度对误报和漏报的平衡影响
- 阈值松紧对 Precision/Recall 的影响
- 持续长度约束对持续故障检测鲁棒性的影响

## 6. 输出文件规划

计划生成以下输出：

- `test/run_mra_experiments.py`
  - 批量组织并执行所有实验
- `test/results/ablation_metrics.csv`
  - 消融实验汇总表
- `test/results/sensitivity_metrics.csv`
  - 参数敏感性实验汇总表
- `test/results/<experiment_name>/config.json`
  - 单实验配置快照
- `test/results/<experiment_name>/metrics.json`
  - 单实验指标
- `test/results/<experiment_name>/scores.npz`
  - 单实验分数与预测结果
- `test/results/<experiment_name>/anomaly_scores.png`
  - 单实验异常分数可视化

## 7. 实施顺序

1. 重构 `mra.py`，将模型结构、损失函数、训练流程、检测参数改为可配置。
2. 保留原有基线训练入口，避免破坏现有使用方式。
3. 编写 `test/run_mra_experiments.py`，支持按组运行：
   - `ablation`
   - `sensitivity`
   - `all`
4. 先验证基线与单个消融实验可跑通，再执行完整实验矩阵。
5. 汇总生成 CSV 和每个实验目录下的结果文件。

## 8. 完成标准

当满足以下条件时视为完成：

- `test/task.md` 明确给出实验设计
- `mra.py` 可通过配置切换不同模块与参数
- 可通过脚本批量运行消融实验和参数敏感性分析
- 结果文件能够自动保存到 `test/results/`
- 至少完成一轮基线验证，并具备完整实验执行能力
