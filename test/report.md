# MRA 模型消融实验与参数敏感性分析报告

## 1. 实验目的

本报告围绕 [mra.py](/home/akira/codespace/mra-detection/mra.py) 中实现的 AGF-ADNet 模型展开，目标包括：

1. 通过消融实验分析模型关键模块对持续故障检测性能的贡献。
2. 通过参数敏感性分析评估模型对结构参数、训练参数和检测后处理参数变化的鲁棒性。
3. 为后续模型裁剪、参数调优和论文撰写提供依据。

## 2. 数据集与任务说明

实验使用的数据如下：

- 训练集：`data/train/train_1.csv`，共 4100 行，18 个变量
- 测试集：`data/test/test_C5_1.csv`，共 4100 行，18 个变量

任务形式为基于重构误差的异常检测。模型先在训练集上学习正常工况的重构规律，再在测试集上计算异常分数，通过阈值与持续报警规则识别故障。按照当前代码设定，测试集前 2000 个窗口视为正常样本，后 2000 个窗口视为异常样本。

## 3. 实验设置

### 3.1 统一设置

- 滑窗起点与采样逻辑：沿用 [mra.py](/home/akira/codespace/mra-detection/mra.py) 中现有实现
- 默认窗口长度：`seq_len=50`
- 标准化方式：先将缺失值置零，再进行 `StandardScaler` 标准化
- 训练轮数：`epochs=3`
- 批大小：`batch_size=32`
- 默认学习率：`lr=1e-3`
- 默认检测参数：
  - `EWMA alpha=0.02`
  - `threshold_std=1.5`
  - `min_run=150`

### 3.2 对比口径说明

- 消融实验采用 3 个随机种子 `(40, 41, 42)` 的集成结果，主用于评估模块贡献。
- 训练型敏感性分析采用单种子 `40` 扫描，目的是控制计算量并观察参数变化趋势。
- 因此，训练型敏感性中的 `seq_len=50`、`d_model=64`、`lr=1e-3` 与消融基线 `full` 不能直接按绝对值横向比较，更适合做同组内趋势分析。

## 4. 评价指标

每个实验统一统计：

- 原始点级指标：Accuracy、Precision、Recall、F1
- 持续故障指标：Accuracy、Precision、Recall、F1
- 异常分数统计：训练集均值/标准差、测试集均值/标准差
- 阈值与报警数量：原始阈值、平滑阈值、点报警数、持续报警数

其中，持续故障 `F1` 作为本报告的主指标。

## 5. 基线结果

完整模型 `full` 的结果如下：

| 指标 | 数值 |
| --- | ---: |
| Raw Accuracy | 0.9025 |
| Raw Precision | 0.8997 |
| Raw Recall | 0.9060 |
| Raw F1 | 0.9028 |
| Persistent Accuracy | 0.9893 |
| Persistent Precision | 1.0000 |
| Persistent Recall | 0.9785 |
| Persistent F1 | 0.9891 |

可以看到，模型在加入 EWMA 平滑和持续长度约束后，持续故障检测性能显著高于点级原始检测性能，说明后处理规则在该数据集上非常有效。

## 6. 消融实验结果与分析

### 6.1 实验结果

| 实验 | Persistent F1 | 相对 `full` 变化 | Persistent Recall | Raw F1 |
| --- | ---: | ---: | ---: | ---: |
| full | 0.9891 | 0.0000 | 0.9785 | 0.9028 |
| no_graph | 0.9881 | -0.0010 | 0.9765 | 0.9144 |
| no_gcn | 0.9889 | -0.0003 | 0.9780 | 0.8857 |
| no_tcn | 0.9896 | +0.0005 | 0.9795 | 0.9173 |
| no_freq | 0.9896 | +0.0005 | 0.9795 | 0.9048 |
| mean_fusion | 0.9465 | -0.0426 | 0.8985 | 0.8419 |
| no_sampling_embed | 0.9468 | -0.0423 | 0.8990 | 0.8482 |
| recon_only_loss | 0.9891 | 0.0000 | 0.9785 | 0.9171 |

### 6.2 结果分析

#### 1. 门控融合与采样率嵌入最关键

`mean_fusion` 和 `no_sampling_embed` 的持续故障 `F1` 均下降约 0.042，且 `Recall` 从 0.9785 下降到约 0.899，说明：

- 时间域与频域信息不能简单平均，门控融合能够更有效地选择不同分支的贡献。
- 采样率类型嵌入对该多速率场景是重要信息来源，去掉后模型对异常模式的时序结构刻画明显变弱。

这两项是当前模型中最值得保留的设计。

#### 2. 图学习、GCN、TCN 和频域分支的影响相对较小

`no_graph`、`no_gcn`、`no_tcn`、`no_freq` 的持续故障 `F1` 与基线差距都很小，甚至 `no_tcn`、`no_freq` 略优于 `full`。这说明：

- 在当前数据集和训练轮数下，时间分支与频率分支之间存在一定冗余；
- 自适应图学习虽然有理论价值，但在当前场景下尚未表现出明显的增益；
- TCN 和频域补全模块对点级指标有一定帮助，但未显著拉开持续故障指标。

这也提示当前任务可能更依赖后处理策略和时序编码，而不是单个局部结构模块。

#### 3. 联合损失对持续故障指标影响有限

`recon_only_loss` 与 `full` 的持续故障 `F1` 完全相同，且 `Raw F1` 还有所提升，表明：

- 频域损失与图稀疏正则在当前短训练设置下，并没有稳定带来收益；
- 当前检测流程中，后处理规则对最终持续故障结果的影响大于联合损失本身。

这并不意味着联合损失无效，更可能说明在 3 个 epoch 的快速实验设置下，其优势还没有充分释放。

## 7. 参数敏感性分析

### 7.1 训练型参数敏感性

#### 7.1.1 窗口长度 `seq_len`

| 参数 | Persistent F1 | Persistent Recall | Raw F1 |
| --- | ---: | ---: | ---: |
| 30 | 0.9922 | 0.9845 | 0.9085 |
| 50 | 0.9570 | 0.9175 | 0.8441 |
| 70 | 0.9741 | 0.9495 | 0.9212 |

分析：

- `seq_len=30` 表现最好，说明较短窗口更适合当前故障模式，可能因为异常信号更局部、更集中。
- `seq_len=70` 好于 `50`，说明更长窗口并非完全无益，但会增加建模难度。
- 当前默认 `seq_len=50` 并不是单种子条件下的最优选择。

#### 7.1.2 Transformer 隐空间维度 `d_model`

| 参数 | Persistent F1 | Persistent Recall | Raw F1 |
| --- | ---: | ---: | ---: |
| 32 | 0.9879 | 0.9760 | 0.9187 |
| 64 | 0.9570 | 0.9175 | 0.8441 |
| 96 | 0.9567 | 0.9170 | 0.8094 |

分析：

- `d_model=32` 优于 `64` 和 `96`，说明当前数据规模下较小的表示维度更稳健。
- 随着 `d_model` 增大，`Raw F1` 持续下降，可能出现过参数化或训练不足问题。
- 当前默认 `d_model=64` 不是最优点，后续可优先尝试较小维度。

#### 7.1.3 学习率 `lr`

| 参数 | Persistent F1 | Persistent Recall | Raw F1 |
| --- | ---: | ---: | ---: |
| 5e-4 | 0.9570 | 0.9175 | 0.8701 |
| 1e-3 | 0.9570 | 0.9175 | 0.8441 |
| 2e-3 | 0.9907 | 0.9815 | 0.8245 |

分析：

- `lr=2e-3` 的持续故障 `F1` 最优，说明更大的学习率在当前 3 个 epoch 的设置下反而更有利于快速收敛到可分离的异常分数。
- 但其 `Raw F1` 下降，说明高学习率下模型更依赖后处理策略，而点级分数本身并不一定更平滑。
- 如果后续继续延长训练轮数，需要重新验证高学习率是否仍然最优。

### 7.2 检测参数敏感性

#### 7.2.1 EWMA 平滑系数 `alpha`

| 参数 | Persistent F1 | Persistent Recall |
| --- | ---: | ---: |
| 0.01 | 0.9848 | 0.9700 |
| 0.02 | 0.9891 | 0.9785 |
| 0.05 | 0.8910 | 0.8035 |

分析：

- `alpha=0.02` 是较优平衡点。
- `alpha=0.05` 下降非常明显，主要表现为 `Recall` 大幅降低，说明平滑过强会掩盖故障上升边界，导致漏检。
- `alpha=0.01` 虽然也可用，但略逊于 `0.02`。

#### 7.2.2 阈值系数 `threshold_std`

| 参数 | Persistent F1 | Persistent Recall |
| --- | ---: | ---: |
| 1.0 | 0.9909 | 0.9820 |
| 1.5 | 0.9891 | 0.9785 |
| 2.0 | 0.9490 | 0.9030 |

分析：

- `threshold_std=1.0` 略优于默认的 `1.5`，说明当前训练分数与测试分数之间有足够间隔，可以使用更积极的阈值。
- `threshold_std=2.0` 时性能明显下降，说明阈值过高会带来大量漏报。
- 在本数据集上，阈值偏保守并不是更好的选择。

#### 7.2.3 持续报警长度 `min_run`

| 参数 | Persistent F1 | Persistent Precision | Persistent Recall |
| --- | ---: | ---: | ---: |
| 100 | 0.9588 | 0.9400 | 0.9785 |
| 150 | 0.9891 | 1.0000 | 0.9785 |
| 200 | 0.9891 | 1.0000 | 0.9785 |

分析：

- `min_run=100` 时 `Recall` 不变，但 `Precision` 从 1.0 降到 0.94，说明过短的持续约束会把短时误报也保留下来。
- `150` 和 `200` 表现一致，说明只要持续长度阈值达到一定水平，模型已经能稳定抑制孤立噪声报警。
- 当前默认 `min_run=150` 是合理的折中选择。

## 8. 综合结论

结合消融和敏感性分析，可以得到以下结论：

1. 当前模型中最关键的设计是门控融合和采样率嵌入，它们对持续故障 `Recall` 和最终 `F1` 影响最大。
2. 图学习、GCN、TCN、频域分支和联合损失在本数据集上的增益有限，说明模型整体性能更可能由多速率编码与检测后处理主导。
3. 检测参数对最终结果影响非常大，尤其是 `alpha` 和 `threshold_std`。不合理的后处理设置会显著降低 `Recall`。
4. 在当前快速实验设置下，较短窗口 `seq_len=30`、较小表示维度 `d_model=32` 和较大学习率 `lr=2e-3` 展现出更好的趋势，值得在后续更充分训练下继续验证。

## 9. 建议

后续建议优先做以下工作：

1. 以 `seq_len=30`、`d_model=32`、`lr=2e-3` 为候选配置，使用多种子和更多 epoch 进行复验。
2. 保留门控融合与采样率嵌入，不建议轻易移除。
3. 若需要压缩模型，可优先尝试弱化图学习、GCN 或频域支路，而不是删除融合与采样率编码。
4. 将检测参数纳入正式调参流程，尤其重点关注 `alpha=0.02` 和 `threshold_std=1.0~1.5` 区间。

## 10. 实验局限性

本次实验仍存在以下限制：

- 训练轮数仅为 3，属于快速验证设置；
- 训练型敏感性分析采用单种子，结论更偏向趋势判断；
- 仅使用一个训练集与一个测试故障场景，泛化结论仍需更多故障类型验证；
- 当前评价更强调持续故障检测，点级异常分数质量仍可继续优化。
